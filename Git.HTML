Comandos Git Bash

Consola

ls -l En lista los archivos del directorio en el que nos encontremos.

clear limpia la pantalla.

cd Abrimos directorios. Ejemplo: cd ProyectoGit

cd .. Retrocedemos un directorios.

pwd Nos muestra la ruta del directorio.

code Git.html Abre el archivo de texto en visual studio code.

cat Git.html Nos deja visualizar el contenido del documento.

mkdir Crea una carpeta en el directorio. Ejemplo: mkdir ProyectoGit

touch Crea un documento en el directorio. Ejemplo: touch Git.html

history Nos muestra todos los comandos utilizados

Git

git init es el comando que activa git en nuestro proyecto creando un espacio en memoria RAM llamado staging y una carpeta .git.

Este comando se encargará de dos cosas: primero, crear una carpeta .git, donde se guardará toda la base de datos con cambios atómicos de nuestro proyecto; segundo, crear un área que conocemos como staging, que guardará temporalmente nuestros archivos (cuando ejecutemos un comando especial para eso) y nos permitirá, más adelante, guardar estos cambios en el repositorio (también con un comando especial).

git status Evidencia el estado de los documentos.

git add Prepara el documento o los documentos (staging) para ser agregados al repositoriol. git add . agrega todos los documentos. Ejemplo: git add Git.html

git commit -m "Primnera versiòn" Se agrega al repositorio el documento o los documentos. Ejemplo: git commit -m "Mensaje"

git show Te muestro los cambios del archivo. Ejemplo: git show Git.html

git log Muestra la historia del archivo.

Staging El área de staging se puede ver como un limbo donde nuestros archivos están por ser enviados al repositorio o ser regresados a la carpeta del proyecto.

ESC + Shit + ZZ Para salir.

git diff Para ver los cambios entre veviones.
